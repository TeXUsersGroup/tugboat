Title: Typesetting external program code and its output: hvextern

Summary: powerful methods of running general subprograms and typesetting their output.

Full text of article: publicly available now.

Author: Herbert Voß

Publication: TUGboat volume 43, number 3 (2022), pages 280-292

DOI (this page): 10.47397/tb/43-3/tb135voss-extern
(previous doi - next doi)

Categories: LaTeX - Software & Tools

Difficulty: Intermediate Plus

Abstract: When writing a book with a mathematical, scientific or technical background, the output of programs is often inserted as text or an illustration; in many cases, also with complete or partial indication of the respective source code. As an author, you have the problem of keeping such external sample programs in sync with the current manuscript. If you keep the source code in the book manuscript itself, and create the external examples at the same time as typesetting the main document, you can be sure the code and output stay consistent.
