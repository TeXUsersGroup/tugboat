Title: Making BachoTeX proceedings—extended version

Summary: autogenerating tables of contents and more using Scheme, Ruby, make, etc.

Full text of article: available to TUG members (join TUG); will be publicly available after the next issue is published.

Author: Jean-Michel Hufflen

Publication: TUGboat volume 45, number 2 (2024), pages 257–263

DOI (this page): 10.47397/tb/45-2/tb140hufflen-bachoproc
(previous doi - next doi)

Category: Publishing

Difficulty: Advanced

Abstract: We report an experiment of making the proceedings of a conference by automatically generating as much information as possible. For example, we look for titles and other metadata in successive source files to build the table of contents. Separate source files must be processed by a TeX-like format, which may be pdfLaTeX, XeLaTeX, LuaLaTeX or ConTeXt. Installing our functions requires a Unix-like make command and some programs written mostly using Scheme, partly using Ruby.
